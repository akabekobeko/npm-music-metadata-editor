## サマリー

致命的な脆弱性は **検出されず**。本ブランチの差分は `packages/gui` の Spreadsheet コンポーネント周辺の **純粋な CSS / レイアウト リファクタ** で、IPC チャネル / ファイル I/O / shell 起動 / 動的コード実行 / 認証 経路のいずれにも触れていない。

差分の中身:

- `Spreadsheet.tsx`: ヘッダーへの `rows` prop 受け渡しを廃止 (連鎖して下流の `useSpreadsheetHeader` / `renderHeader` が rows を要求しなくなる)
- `SpreadsheetBody.tsx`: `<tr>` の `w-full` を撤去し `<td>` 側に `border-b` を移設、`column.sticky === "left"` 判定の class を削除
- `SpreadsheetHeader.tsx`: `<tr>` の `border-b` を `<th>` 側に移設、`column.sticky === "left"` 判定の class を削除、`rows` prop の受け取りを廃止
- `SpreadsheetHeader/FileNameHeaderCell.tsx`: 削除 (件数表示は Header / StatusBar の `header.fileCount.*` 翻訳キーで既出のため重複機能)
- `SpreadsheetHeader/renderHeader.tsx`: `fileName` 列の特殊ケース廃止、すべての列を共通の `<span className="font-medium">{column.title}</span>` で描画
- `SpreadsheetHeader/useSpreadsheetHeader.ts`: 不要になった `rows` 引数を撤去
- `features/spreadsheet/constants.ts`: `fileName` 列定義から `sticky: "left"` を撤去
- `features/spreadsheet/types.ts`: `ColumnDefinition.sticky?: "left"` フィールドを撤去 (用途が無くなったため)
- `features/spreadsheet/buildColumns.ts`: docstring の "pinned to the left" 説明を "prepended" に更新
- `features/spreadsheet/buildColumns.test.ts`: sticky アサーションを削除

## 良い点 (Strengths)

v0.0.0 → v0.1.0 → v0.5.0 (前回 `fix/spreadsheet-editing-cell-click`) で確立した防御線をすべて踏襲。本ブランチの CSS リファクタは構造的にも以下の点でプラス:

1. **`webPreferences.sandbox: true` の維持 (継続)** — `packages/gui/src/main/main.ts`。Renderer サンドボックスの境界線は維持。
2. **`expandDroppedPaths` の三重防御 (継続)** — `lstat` (symlink を辿らない) / 深さ 3 制限 / 拡張子 allowlist は変更なし。
3. **`webUtils.getPathForFile` 経由の D&D path 解決 (継続)** — preload は byte 単位で同一なので Renderer が `File` を偽造して絶対パスを得る経路は引き続き無い。
4. **IPC `MenuAction` の閉じた string union (継続)** — `packages/gui/src/main/ipc/types.ts`。新規 IPC チャネル / action の追加なし。
5. **`shell.openExternal` の URL がハードコード (継続)** — Renderer 由来の URL 流入経路は無い。
6. **ログ転送が出力先プロセス内のみ (継続)** — `onLogForward` / `onFatalReport` は外部 echo back しない。
7. **`mergeSettings` の明示フィールド戦略 (継続)** — `theme` / `locale` は scalar のみ採用。
8. **`resolveLocale` の allowlist (継続)** — `SUPPORTED_LOCALES` (`["en", "ja"]`) との一致比較で safe fallback。
9. **ランタイム依存ゼロ ポリシー (継続)** — 本ブランチでも追加なし。Main は `electron` + `@akabeko/music-metadata-editor`、Renderer は React 系のみ。
10. **`@mme/ipc` 仮想モジュールが型のみ (継続)** — `import type` 限定で TypeScript が elide するため renderer bundle に main の value module は到達しない。
11. **`pnpm-workspace.yaml` の `allowBuilds` map (継続)** — postinstall 実行は明示許可した 4 パッケージのみ。
12. **`useEditableCell` / `EditableCell` の commit pipeline は不変 (継続)** — 前回 `fix/spreadsheet-editing-cell-click` で導入された `validateTagValue` 共有経路は本ブランチで触っていない。Enter / blur 双方の commit が同一バリデータを通る構造は維持。
13. **`renderHeader` の特殊ケース削除で attack surface が縮小 (本ブランチ新規)** — `packages/gui/src/renderer/components/app/Spreadsheet/SpreadsheetHeader/renderHeader.tsx`。`column.id === "fileName"` での専用コンポーネント呼び出しを廃止し、すべての列を `<span className="font-medium">{column.title}</span>` で統一。`column.title` は `COLUMN_REGISTRY` のハードコード文字列のみで Renderer 内に外部入力経路は無く、React の自動エスケープを介する経路に統一されたことで分岐ごとの差異リスクが消滅。
14. **`useSpreadsheetHeader` から `rows` を撤去 (本ブランチ新規)** — `packages/gui/src/renderer/components/app/Spreadsheet/SpreadsheetHeader/useSpreadsheetHeader.ts`。ヘッダー描画の memo 依存配列が `[columns]` のみとなり、`rows` 変更時の不要な再計算が消える。情報経路の縮小はそのまま侵入面の縮小でもある (rows = ファイルパス由来の文字列を含む TrackRow 配列で、ヘッダー文字列描画には不要な情報だった)。
15. **`SpreadsheetBody.tsx` の `<tr>` から `w-full` を外しても absolute + flex で auto width がコンテンツ幅 = `<td>` 幅合算となる (本ブランチ新規)** — `<tr>` の `position: absolute` 配下で `flex` の auto width は子要素 (`shrink-0` + 明示 `width`) の総和に解決される。横スクロール ハンドリングは `<table>` の `<col>` 列幅で決まる thead 経由で発生し、絶対配置の tbody 行は scroll 駆動側にはならない。レイアウトは `<col>` ベースの thead が支配する従来構造のまま。
16. **`column.sticky` 関連 CSS / 型 / フィールドを完全削除 (本ブランチ新規)** — `ColumnDefinition.sticky?` 型の削除、`COLUMN_REGISTRY.fileName.sticky` の削除、`SpreadsheetHeader` / `SpreadsheetBody` の `column.sticky === "left"` 分岐削除を一括して行うことで、「型は残っているが使われていない」「定義は消えたが CSS だけ残る」といった不整合を残さない。dead code 経由の混乱・誤読リスクを構造的に排除。

## 所見 (Findings)

### `feat/gui-improve-cell-styles` 差分起因の新規 Finding

**該当なし**。本ブランチは React TSX / Tailwind class / TypeScript 型定義の編集に閉じており、IPC チャネル / ファイル I/O / 動的コード実行 / shell 起動 / 認証 / 暗号 / ナビゲーション制御のいずれにも触れていない。

検証済の保留 (false positive 判定):

- **`renderHeader` で文字列を直接描画していることに伴う XSS 懸念**: `column.title` は `COLUMN_REGISTRY` (`packages/gui/src/renderer/features/spreadsheet/constants.ts`) のハードコード値のみで、外部入力で動的に変わる経路は無い。React は `{column.title}` を text node としてエスケープするため、仮に `title` が動的になっても DOM 注入経路は塞がっている (HARD EXCLUSION 6 / 8 にも該当)。
- **`<tr>` から `w-full` を外したことで思わぬ overlap が起きる懸念**: `<tr>` は `position: absolute`、子の `<td>` は `flex shrink-0` + 明示 `width`。auto width はコンテンツ幅合算に解決され、複数行の重なりは `transform: translateY(...)` の段で row ごとに上下に分離される。stacking context / clickjacking の文脈で UI redress 攻撃に転用できる新規 overlay は導入されていない。Electron renderer 内での DOM 操作はそもそも信頼境界の内側。
- **`FileNameHeaderCell.tsx` 削除に伴う情報露出変化**: 削除前は `(N files)` を File 列ヘッダーに表示していたが、同じ件数は `Header.tsx` および `StatusBar.tsx` で `header.fileCount.*` 翻訳キー経由で既に表示されていた (`packages/gui/src/renderer/components/app/AppShell/Header.tsx:103`, `StatusBar.tsx:35`)。情報量は減少方向のみで増加方向ではない (PII 等の新規漏出経路は無い)。
- **`buildColumns.test.ts` の `sticky` アサーション削除がテスト カバレッジを下げる懸念**: テスト ファイル (HARD EXCLUSION 11) の話で production code path には到達しない。`sticky` フィールド自体が型 / 定数から完全削除されているため、アサーション自体が型エラーになる。

### v0.1.0 から継続している LOW (`feat/gui-improve-cell-styles` で悪化なし)

- **LOW-2 — Content-Security-Policy が未設定 (継続)**: `packages/gui/src/renderer/index.html`。本ブランチでも未対処。React 経由で文字列はエスケープされるため XSS 経路は依然として無いが、二段目の防御として `default-src 'self'` ベースの CSP を `<meta>` か `onHeadersReceived` で適用するのが望ましい。Vite dev サーバーが `'unsafe-eval'` を要求するため production のみ適用する分岐が必要。
- **LOW-3 — クリップボード ペーストにセーフ ガードが無い (継続)**: `packages/gui/src/renderer/components/app/Spreadsheet/useKeyboard.ts` の `dispatchPaste`。Phase 8 で改行を含む列 (`Lyrics`) と payload が opaque な列 (`Pictures`) の列方向 broadcast 経路は型レベルで塞がれているが、「短文系列を列選択して 1 値ペースト → 全行 fill」のケース (Phase 4 broadcast 規則の本来動作) で、`applyPaste` 前段に「行数 > rows.length」「セル長 > N KiB」の確認ダイアログを挟む価値が引き続き残る。本ブランチではキーボード / ペースト ハンドラ非変更。
- **LOW-4 — `onSaveTrack` の許可リスト未実装 (継続)**: `packages/gui/src/main/ipc/onSaveTrack.ts`。本ブランチでは Main / IPC ハンドラに変更が無いため、`filePath` 経路の数 (dialog / recentFiles / D&D / Open Recent) は v0.1.0 と同じ。

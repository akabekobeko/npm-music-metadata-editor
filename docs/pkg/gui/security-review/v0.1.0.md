## 良い点 (Strengths)

v0.0.0 → v0.1.0 (Phase 8 / refactor/gui) で確立した防御線をすべて踏襲。本 PR の依存ツール / バージョン更新は構造的にも以下の点でプラス:

1. **`webPreferences.sandbox: true` の維持** — `packages/gui/src/main/main.ts`。Renderer サンドボックスの境界線は維持。
2. **`expandDroppedPaths` の三重防御 (継続)** — `lstat` (symlink を辿らない) / 深さ 3 制限 / 拡張子 allowlist は変更なし。
3. **`webUtils.getPathForFile` 経由の D&D path 解決 (継続)** — preload は byte 単位で同一なので Renderer が `File` を偽造して絶対パスを得る経路は引き続き無い。
4. **IPC `MenuAction` の閉じた string union (継続)** — `packages/gui/src/main/ipc/types.ts`。新規 IPC チャネル / action の追加なし。
5. **`mme:menu:setState` の payload が data のみ (継続)** — `applyMenuStateSnapshot` の構造化クローン経路は変更なし。
6. **`shell.openExternal` の URL がハードコード (継続)** — Renderer 由来の URL 流入経路は無い。
7. **ログ転送が出力先プロセス内のみ (継続)** — `onLogForward` / `onFatalReport` は外部 echo back しない。
8. **`setupFatalHandlers` の plain object payload (継続)** — `Error.cause` 等の native chain は剥がれる。
9. **`mergeSettings` の明示フィールド戦略 (継続)** — `theme` / `locale` は scalar のみ採用。
10. **`resolveLocale` の allowlist (継続)** — `SUPPORTED_LOCALES` (`["en", "ja"]`) との一致比較で safe fallback。
11. **ランタイム依存ゼロ ポリシー (継続)** — 本 PR でも追加なし。Main は `electron` + `@akabeko/music-metadata-editor`、Renderer は React 系のみ。
12. **`useDragAndDrop` の `event.preventDefault` (継続)** — Chromium の page navigation 抑止。
13. **`ColumnDefinition.selectable` 導入による列ペースト経路の構造的封じ (継続)** — Phase 8 で確立した「Lyrics / Pictures は列選択不可」の不変条件は維持。
14. **`@mme/ipc` 仮想モジュールが型のみ (継続)** — `import type` 限定で TypeScript が elide するため renderer bundle に main の value module は到達しない。
15. **`shared/locales/` の renderer / main 越境解消 (継続)** — `tsconfig.web.json` / `tsconfig.node.json` の include 整理は維持。
16. **`onlyBuiltDependencies` を `pnpm-workspace.yaml` へ集約 (本 PR 新規)** — root `package.json` の `pnpm.onlyBuiltDependencies` を `pnpm-workspace.yaml` の `onlyBuiltDependencies` に一元化。pnpm 11 の推奨配置に揃え、workspace 全体で参照される **明示的な許可リスト** で postinstall 実行可能パッケージを制御する形を維持。pnpm 11 の既定はインストール スクリプトを **拒否** (ERR_PNPM_IGNORED_BUILDS) で、許可リストに無いパッケージの任意コードは実行されない。今回追加された `electron-winstaller` (electron-builder の Windows squirrel ビルダー間接依存) と `msw` (vitest peer / shadcn 経由) は警告抑止のための明示登録で、いずれも npm 上の実在パッケージ (検証済み発行者) からの配布。
17. **pnpm 11 への移行 (本 PR 新規)** — `corepack` で固定された `packageManager` フィールドを `pnpm@11.0.9` に上げ、`.mise.toml` (root / gui) も同一バージョンへ追従。チェックサムを伴う `pnpm-lock.yaml` は再生成されており、各依存の resolved tarball のハッシュは pnpm 11 の検証経路で再評価されている。
18. **devDependencies のマイナー / パッチ更新 (本 PR 新規)** — `electron` 41.3.0 → 41.5.0、`react` / `react-dom` 19.2.5 → 19.2.6、`vite` 8.0.10 → 8.0.11、`shadcn` 4.6.0 → 4.7.0、`@types/node` 25.6.0 → 25.6.2、`@biomejs/biome` 2.4.13 → 2.4.14。いずれも SemVer の patch / minor で破壊的変更を伴わず、Electron の Chromium / V8 セキュリティ パッチが取り込まれる方向の更新。

## 所見 (Findings)

### `feat/bump-pnpm-and-deps` 差分起因の新規 Finding

**該当なし**。本 PR は manifests + lockfile + ツール バージョン管理ファイルのみを変更しており、`packages/*/src` は無変更。IPC ハンドラ / ファイル I/O / 動的コード実行 / 認証 / 暗号 / shell 起動 / ナビゲーション制御のいずれにも触れていない。

検証済の保留 (false positive 判定):

- **`onlyBuiltDependencies` に `electron-winstaller` / `msw` を追加することで RCE サーフェスが広がる懸念**: pnpm 11 の既定はビルド スクリプト全拒否 (`ERR_PNPM_IGNORED_BUILDS`) で、許可リストへの追加はあくまで npm パッケージの postinstall を **明示的に実行許可** する操作。`electron-winstaller` は `electron-builder` の Windows ビルド経路に必要 (squirrel-windows の native bits 配置)、`msw` は vitest peer の Service Worker セットアップに必要で、いずれも開発 / ビルド時のみに走り production 配布物には影響しない。upstream `electron-starter#26` と同じ方針。
- **pnpm 11 メジャー アップ自体に伴う lockfile checksum 形式変更の懸念**: pnpm-lock.yaml v9 のままで pnpm 11 が再書き出しした形 (フォーマットは v9 互換)。`integrity:` フィールドは依然 `sha512-...` 形式で各 tarball の sha512 を検証する経路は維持されている。
- **`electron` 41.3.0 → 41.5.0 で Chromium のメジャーが変わる懸念**: 41 系はそのまま (`41.3` → `41.5` は Electron の minor)。Chromium / V8 / Node のメジャー版は変わらず、`webPreferences.sandbox: true` 等の API 互換は維持。
- **`react` / `react-dom` 19.2.5 → 19.2.6 で hydration 経路が変わる懸念**: SemVer の patch リリースで API / hydration 仕様には変更無し。
- **`shadcn` 4.6.0 → 4.7.0 が CLI として postinstall を持つ懸念**: shadcn は CLI ツール (`pnpm shadcn` 経由で生成スクリプトを走らせる) で本リポでは `dependencies` ではなく `devDependencies`。生成済みの UI コードは `packages/gui/src/renderer/components/ui/**` に直接コミットされており runtime には CLI は含まれない。

### v0.1.0 から継続している LOW (`feat/bump-pnpm-and-deps` で悪化なし)

- **LOW-2 — Content-Security-Policy が未設定 (継続)**: `packages/gui/src/renderer/index.html`。本 PR でも未対処。React 経由で文字列はエスケープされるため XSS 経路は依然として無いが、二段目の防御として `default-src 'self'` ベースの CSP を `<meta>` か `onHeadersReceived` で適用するのが望ましい。Vite dev サーバーが `'unsafe-eval'` を要求するため production のみ適用する分岐が必要。
- **LOW-3 — クリップボード ペーストにセーフ ガードが無い (継続)**: `packages/gui/src/renderer/components/app/Spreadsheet/useKeyboard.ts` の `dispatchPaste`。Phase 8 で改行を含む列 (`Lyrics`) と payload が opaque な列 (`Pictures`) の列方向 broadcast 経路は型レベルで塞がれているが、「短文系列を列選択して 1 値ペースト → 全行 fill」のケース (Phase 4 broadcast 規則の本来動作) で、`applyPaste` 前段に「行数 > rows.length」「セル長 > N KiB」の確認ダイアログを挟む価値が引き続き残る。
- **LOW-4 — `onSaveTrack` の許可リスト未実装 (継続)**: `packages/gui/src/main/ipc/onSaveTrack.ts`。本 PR では Main / IPC ハンドラに変更が無いため、`filePath` 経路の数 (dialog / recentFiles / D&D / Open Recent) は v0.1.0 と同じ。

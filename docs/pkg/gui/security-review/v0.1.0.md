## サマリー

致命的な脆弱性は **検出されず**。`refactor/gui` の差分は **すべて以下のいずれか** で、攻撃面の追加・既存防御の弱体化のいずれも発生していない:

- React hook の機械的抽出 (`useState` / `useEffect` 等を `use<Component>.ts` へ移動)
- ディレクトリ再編 (`SpreadsheetHeader/` / `SpreadsheetBody/` への集約、`fileName/` の解体、`src/main/locales/` → `src/shared/locales/`)
- Object 型のプロパティ単位 TSDoc 追加 (約 190 件)
- IPC 契約型の renderer 側経路を `vite-env.d.ts` の `declare module "@mme/ipc"` へ集約 (型のみ、ランタイム不変)
- 完了済 Phase に依存していた stale コメントの書き換え
- `useGlobalShortcuts` 削除 (native menu accelerator が `buildAppMenu.ts:86,96` に既存)、`SettingsSnapshot` alias 削除 (純粋な型 alias の解消)

差分には新規 IPC チャネル / 新規 ipcMain ハンドラ / 新規 ファイル I/O / 新規 `dangerouslySetInnerHTML` / 新規 `shell.openExternal` のいずれも含まれない。`packages/gui/src/main/ipc/onSaveTrack.ts` / `onWriteBytes.ts` / `onReadBytes.ts` / `onShowSaveFile.ts` / `dnd/expandDroppedPaths.ts` / `preload/preload.ts` / `settings/settings.ts` は **byte 単位で main と同一**。

## 良い点 (Strengths)

v0.0.0 → v0.1.0 (Phase 8) で確立した防御線をすべて踏襲。リファクタで構造的に強化された箇所もある:

1. **`webPreferences.sandbox: true` の維持** — `packages/gui/src/main/main.ts`。Renderer サンドボックスの境界線は維持。
2. **`expandDroppedPaths` の三重防御 (継続)** — `lstat` (symlink を辿らない) / 深さ 3 制限 / 拡張子 allowlist は変更なし。
3. **`webUtils.getPathForFile` 経由の D&D path 解決 (継続)** — preload は `byte 単位で同一` なので Renderer が `File` を偽造して絶対パスを得る経路は引き続き無い。
4. **IPC `MenuAction` の閉じた string union (継続)** — `packages/gui/src/main/ipc/types.ts`。新規 IPC チャネル / action の追加なし。
5. **`mme:menu:setState` の payload が data のみ (継続)** — `applyMenuStateSnapshot` の構造化クローン経路は変更なし。
6. **`shell.openExternal` の URL がハードコード (継続)** — Renderer 由来の URL 流入経路は無い。
7. **ログ転送が出力先プロセス内のみ (継続)** — `onLogForward` / `onFatalReport` は外部 echo back しない。
8. **`setupFatalHandlers` の plain object payload (継続)** — `Error.cause` 等の native chain は剥がれる。
9. **`mergeSettings` の明示フィールド戦略 (継続)** — `theme` / `locale` は scalar のみ採用。
10. **`resolveLocale` の allowlist (継続)** — `SUPPORTED_LOCALES` (`["en", "ja"]`) との一致比較で safe fallback。`shared/locales/` 移転後も resolveLocale 関数本体は byte 単位で同一。
11. **ランタイム依存ゼロ ポリシー (継続)** — リファクタで依存追加なし。Main は `electron` + `@akabeko/music-metadata-editor`、Renderer は React 系のみ。
12. **`useDragAndDrop` の `event.preventDefault` (継続)** — Chromium の page navigation 抑止。
13. **`ColumnDefinition.selectable` 導入による列ペースト経路の構造的封じ (継続)** — Phase 8 で確立した「Lyrics / Pictures は列選択不可」の不変条件は本リファクタでも維持 (`features/spreadsheet/constants.ts` / `useSelection.ts` / `useKeyboard.ts`)。
14. **`isColumnSelectable` が純関数 (継続)** — `packages/gui/src/renderer/features/spreadsheet/isColumnSelectable.ts`。1 行純関数のままで I/O も副作用も無い。
15. **`<th>` 属性への文字列補間が信頼境界の内側のみ (継続)** — `SpreadsheetHeader/SpreadsheetHeader.tsx` 移動後も `column.title` は `COLUMN_REGISTRY` ハードコード文字列由来で外部入力は混入しない。
16. **`@mme/ipc` 仮想モジュールが型のみ (本リファクタ新規)** — `packages/gui/src/renderer/vite-env.d.ts` の `declare module "@mme/ipc" { export * from "../main/ipc/types" }` は ambient 型宣言。すべての利用箇所が `import type` のため TypeScript の elided import として bundler に到達する前に消える。Renderer bundle が main の値モジュールを引き込まないことが構造的に保たれる (renderer が `electron` / `@akabeko/music-metadata-editor` を bundle しないという v0.1.0 の不変条件はそのまま)。
17. **`shared/locales/` 移転で main / renderer 双方の越境が解消 (本リファクタ新規)** — `packages/gui/src/shared/locales/` は `tsconfig.web.json` の include に `src/shared/**` で加わり、`tsconfig.node.json` にも同じく `src/shared/**` を追加。`src/main/**` を renderer の include に含める必要が無くなり、renderer の compile スコープが厳密に narrower になった。locales 配下のファイルは byte 単位で移動 (rename 100%) で挙動変化なし。
18. **`useGlobalShortcuts` 削除でショートカット経路が単一化 (本リファクタ新規)** — Cmd/Ctrl+O / Cmd/Ctrl+S は `packages/gui/src/main/menu/buildAppMenu.ts:86,96` の native accelerator → `mme:menu:action` を経由する経路に集約。document-level keydown listener との二重実行が解消。Renderer 側で重複するハンドラは無くなり、accelerator → menu action subscriber → `useMenuActions` 経由で `useFileOpen` / `save.saveAll` を呼ぶ単一経路となる。
19. **`SettingsSnapshot` alias 削除によるサーフェス縮小 (本リファクタ新規)** — `packages/gui/src/main/ipc/types.ts` の `SettingsSnapshot = AppSettings` を削除し `MmeBridge.settings.get` / `set` および `onGetSettings` / `onSetSettings` の戻り値型を `AppSettings` 直接参照に変更。型 alias の二重表現が解消され IPC 契約サーフェスが単純化 (ランタイム挙動は完全同一)。

## 所見 (Findings)

### `refactor/gui` 差分起因の新規 Finding

**該当なし**。本 PR で変更されたコードは hook 抽出 / ディレクトリ移動 / 型 alias 整理 / 仮想モジュール宣言 / コメント書き換えで、IPC ハンドラ / ファイル I/O / 動的コード実行 / 認証 / 暗号 / shell 起動 / ナビゲーション制御のいずれにも触れていない。

検証済の保留 (false positive 判定):

- **`useGlobalShortcuts` 削除でショートカットが効かなくなる懸念**: native menu accelerator (`buildAppMenu.ts:86,96`) が同一 `mme:menu:action` action を発火させるため UX 上も等価。セキュリティ的には二重実行解消によりむしろ望ましい。
- **`@mme/ipc` 仮想モジュールが ambient 宣言で main 型を露出する懸念**: 宣言は `import type` でしか参照されておらず、TypeScript は elide するため renderer bundle に main の value module は到達しない (`renderer/vite.config.ts` / `vitest.config.ts` のいずれにも `@mme/ipc` の resolver alias は無く、runtime に解決を試みる経路自体が存在しない)。
- **`shared/locales/` 移転で renderer から `src/main/**` のクラスが引き込まれる懸念**: `tsconfig.web.json` の include は `src/main/locales/**` から `src/shared/**` に置換されたのみで、`src/main/**` の他ディレクトリを含めていない。Vite の renderer build は `root: src/renderer/` で動き、`src/shared/**` への相対 import は解決可能だが `src/main/**` への import は依然として renderer のスコープ外。

### v0.1.0 から継続している LOW (`refactor/gui` で悪化なし)

- **LOW-2 — Content-Security-Policy が未設定 (継続)**: `packages/gui/src/renderer/index.html`。本リファクタでも未対処。React 経由で文字列はエスケープされるため XSS 経路は依然として無いが、二段目の防御として `default-src 'self'` ベースの CSP を `<meta>` か `onHeadersReceived` で適用するのが望ましい。Vite dev サーバーが `'unsafe-eval'` を要求するため production のみ適用する分岐が必要。
- **LOW-3 — クリップボード ペーストにセーフ ガードが無い (継続)**: `packages/gui/src/renderer/components/app/Spreadsheet/useKeyboard.ts` の `dispatchPaste` (旧 `Spreadsheet.tsx`)。Phase 8 で改行を含む列 (`Lyrics`) と payload が opaque な列 (`Pictures`) の列方向 broadcast 経路は型レベルで塞がれており、本リファクタでも `useSelection` の `handleColumnHeaderClick` が `isColumnSelectable` で gate する構造は維持。残るは「短文系列を列選択して 1 値ペースト → 全行 fill」のケース (Phase 4 broadcast 規則の本来動作) で、`applyPaste` 前段に「行数 > rows.length」「セル長 > N KiB」の確認ダイアログを挟む価値が引き続き残る。
- **LOW-4 — `onSaveTrack` の許可リスト未実装 (継続)**: `packages/gui/src/main/ipc/onSaveTrack.ts`。本リファクタでは Main / IPC ハンドラに変更が無いため、`filePath` 経路の数 (dialog / recentFiles / D&D / Open Recent) は v0.1.0 と同じ。
